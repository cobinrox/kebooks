Cheatsheet

How To Solution
Download a repo from a remote Git server 

      git clone <remote_repo>
      git cd <repo name>
Get status of a local repo 

      git remote update
      git status
      git show --name-only HEAD
Display available branches 

      git remote update
      git branch -a
Create a new branch 

      git checkout <the parent_for_your_new_branch>
      git pull
      git checkout -b <name_of_your_new_branch>
Checkout a branch First, save/commit/stash your current branch's local changes, if any. Then: 

        git checkout <the branch_you_are_interested_in>
        git pull
Pull latest changes from a branch on the remote repo First, save/commit/stash your current branch's local changes. Then do your pull: 

        git pull <origin> <the branch_you_are_interested_in>
Then resolve any merge conflicts and re-run tests.
Undo a pull 

        git reset --soft HEAD~
        git merge --abort
Then resolve any merge conflicts and re-run tests.
Push a brand-new current branch to remote First, save/commit/stash your current branch's local changes. Then: 

        git push -u origin <your_new_branchs_name>
Push a feature branch to the feature's branch on the remote First, save/commit/stash your current branch's local changes and pull the master into your branch. Then: 

          git pull <origin> <parent_branch_name>
          (resolve any merge conflicts, re-run tests)
          git push
