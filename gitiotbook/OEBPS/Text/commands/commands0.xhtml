Git Commands

This chapter demonstrates the basic workflow of using Git and its basic commands. These are the common commands that you will normally be running in a multi-person team environment, with a remote Git server hosting GitHub, Bitbucket, or something similar.

There are only a handful of commands that you normally run, and you will see them all the time, usually in the order summarized below.

  1. git clone to download the remote repo that you've been assigned
  2. git checkout/checkout -b to get to the particular branch you've been assigned or to create a new branch for your
  3. make your local changes
  4. git add move your changes to the staging area
  5. git commit move your changes to the committed area
  6. git pull optionally, but best practice to download any updates that were made up on the repo
  7. git push move your changes up to the remote repo
  8. wash rinse, repeat: continue to make local changes, stage, commit, pull, and push your branch as you develop day-to-day
  9. when your last change has been made for your work item or feature and pushed, request a "pull" on your branch, i.e. request that your branch be merged into the master (or the develop/sprint branch or whatever parent your branch is a child of)

The "Local/Remote" column is used to distinguish Git commands that work or are affected only by the local repo, on your laptop, versus whether the command works in conjunction with the remote (server/cloud) repo. It is important to understand whether a Git command that you execute either affects or works in conjunction with the remote repo. This is a can be a cause of confusion so it is worthwhile to consider this distinction when you run a Git command.
