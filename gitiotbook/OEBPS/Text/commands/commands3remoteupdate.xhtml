Syntax:


    git remote update

git remote update: What it's supposed to do

This command is your buddy. Sort of. It interacts with the remote repo's ledger and downloads the latest information about the remote repo down to and updates your local ledger. Think of it like having a copy of a newspaper from three days ago that you had delivered to you. Obviously, new things have been going on since the last three days, but your "git status" command will know nothing about the outside world. That is where the "git remote update" command comes in. It will update the ledger or table of contents of your newspaper (not the actual articles) with what is on the remote repo. So, in the George example, running "git remote update" will update your local ledger with new information about changes that have been made on the remote. Again, this is like updating a ledger or table of contents, but does not actually download any file changes to your laptop. That work is done by the "git pull" command, which will will discuss a little later.

An example using this command is below, where, unbeknownst to the user, George has pushed up a change to the branch. The user first runs "git status" which gives them a false sense of security that they seem to be "up-to-date", but then they run "git remote update" which updates the local repo's ledger/card catalog and then finishes with a final "git status" which shows where the user really stands with respect to the remote repo.

git remote update: What they forgot to mention

This is a pretty useful command to actually go out and get a summary list of the changes that have been made up on the remote. You can also use it to merely test if you have network connectivity up to the remote.

Again, when using "git status", be cognizant of what it is not showing you unless you execute a "git remote update" first. Other wise you may be lulled into a false sense of security in thinking that there were no changes made up on the remote that you need to worry about.

git remote update: Bottomline

  • Gets most recent information about the remote repo
  • You can use this to test network connectivity to the Git server
