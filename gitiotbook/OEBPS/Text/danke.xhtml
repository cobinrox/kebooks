This section will walk, in slow motion, a typical Git scenario, one that you may encounter in your job. The scenario set up is:

  • A "sprint/dev" branch called "develop" has already been created by a gargoyle and is available on your company's remote "Constitution" repo, which is stored in a Github server
  • You have been assigned the task of adding the phrase "Do ordain and establish this constitution" to the "Preamble.txt" artifact of the Constitution repo
  • You have VisualStudio Code and SourceTree installed on your laptop
  • You have installed Git (as a client) on your laptop
  • You have a login to the company's Git remote repo

You would normally execute the following steps, based on Git best practices> (see Best Practices chapter):

  1. Open a command-line terminal on your laptop
  2. Create a "gitrepositorys" directory somewhere on your laptop:
          mkdir myStupidGitrepositorys
  3. Download the Constitution repo:
          git clone https://github.com/sepsinod2/constitution.git
  4. Change to the repo's local directory:
          cd constitution
  5. Checkout the sprint/dev branch:
          git checkout dev
  6. Create your own branch:
          git branch -c
