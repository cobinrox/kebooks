GLOSSARY

Term Definition
Artifact An item that is being version controlled, such as a file, source code, a document, etc.
Branch, Develop A "buffer" child branch that is typically used as a child of a master/main branch and as the parent for individual developer branches during a period of time, such as a "sprint". As each developer finishes up their feature branch, it is merged into the develop branch and, at the end of the sprint, the develop branch is then rolled up into the master/main branch. Having a develop branch between a parent and child provides a safety net to help preserve a clean master parent.
Branch, Feature A child branch usually worked by a single person for a single feature or bug.
Branch, Master/Main The primary, yea-verily ground-truth of a repository.
Branch, Parent A branch from which other (child) branches are created and into which those child branches are expected to be merged.
Copy, Local An artifact that is stored on a user's machine/laptop. It usually represents a copy of the master artifact and which a user works on.
Copy, Master/Main An artifact that represents the most-recent or published value. The master copy is stored on a remote repository and may also be backed up to a secondary off-line location.
CM, Configuration Managmement A team that maintains and, sometimes, enforces and implements rules on a set of artifacts.
Merge/Pull Request A capability provided by different Git implementations to notify recipients that a child branch is ready to be merged into a parent branch.
Push To upload artifacts to the remote repository.
QA/Quality Assurance A team or process of checking that a set of artifacts meets expectations. A QA team may also create and execute various types of tests against the artifacts.
Repo/Repository This is where the artifacts are stored. This is usually a file or version control service of some type.
Repository, Local The artifacts that are stored on a user's laptop.
Repository, Remote The main repository where a project or company's files are officially stored. This is usually a large file server running in the cloud or in an IT closet. It is usually backed up nightly and may be backed up to an off-site location.
Test, Regression Usually refers to testing artifacts at a medium-level with previously-developed tests to ensure that the basic artifacts work as expected from the last time a regression test was executed.
Test, Unit Usually refers to testing artifacts at a very low-level to ensure that artifacts work as required. Should include positive and negative testing.
Test, Integration Usually refers to testing artifacts at a high-level from one repo or team with another team or company's artifacts.
