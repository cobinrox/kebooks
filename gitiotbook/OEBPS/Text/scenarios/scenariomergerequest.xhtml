Scenario: Merge/Pull Request

This scenario description gives a little context and detail to a "merge/pull request".

  1. Developer finishes up a feature branch, e.g. Feature X, has downloaded and merged the most-recent parent, and has re-run regression tests.
  2. Developer pushes last changes up to the remote repo.
  3. Developer logs on to the Git management system and creates a merge request for Feature X.
  4. Team members are notified that Feature X is ready to be merged into the parent, e.g. a "develop" branch.
  5. A team lead, configuration manager, QA tester, another team member, or maybe even the original developer, depending on the team's workflow, uses the Git management system to merge the feature branch into the parent.
