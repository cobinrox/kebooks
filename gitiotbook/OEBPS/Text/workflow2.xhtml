Terms

Before presenting the workflows, a few definition/terms are provided below. Note that all of the definitions that are presented in each chapter are duplicated at the end of the book in a Glossary chapter.
Term Definition
Artifact An item that is being version controlled, such as a file, source code, a document, etc.
Master Copy An artifact that represents the most-recent or published value. The master copy is stored on a remote repository and may also be backed up to a secondary off-line location.
Local Copy An artifact that is stored on a user's machine/laptop. It usually represents a copy of the master artifact and which a user works on.
Repository This is where the artifacts are stored. This is usually a file or version control service of some type.
Remote Repository The main repository where a project or company's files are officially stored. This is usually a large file server running in the cloud or in an IT closet. It is usually backed up nightly and may be backed up to an off-site location.
Local Repository The artifacts that are stored on a user's laptop.
Push To upload artifacts to the remote repository.
